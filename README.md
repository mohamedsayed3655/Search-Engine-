## 🔍 Ranking Algorithms

### TF-IDF (Term Frequency – Inverse Document Frequency)
Classic vector-space model using **cosine similarity**.

$$\text{TF-IDF}(t, d) = \frac{f_{t,d}}{\max_f} \times \log\left(\frac{N+1}{df_t+1}\right) + 1$$

- **TF**: How often the term appears in the document (log-normalized)
- **IDF**: How rare the term is across all documents (smoothed)
- **Scoring**: Cosine similarity between query and document vectors
- Best for: **rare, specific keywords**

---

### BM25 (Okapi BM25)
Probabilistic ranking model with **TF saturation** and **document-length normalization**.

$$\text{BM25}(t, d) = \text{IDF}(t) \times \frac{f_{t,d} \cdot (k_1 + 1)}{f_{t,d} + k_1 \cdot \left(1 - b + b \cdot \frac{|d|}{\text{avgdl}}\right)}$$

| Parameter | Value | Description |
|-----------|-------|-------------|
| `k1` | 1.5 | TF saturation — controls how much repeated terms help |
| `b`  | 0.75 | Length normalization — penalizes long documents |

- **IDF**: Robertson-Sparck Jones formula
- Best for: **short, natural language queries**

---

### Hybrid
Weighted combination of both models after **min-max normalization**:

$$\text{Hybrid} = \alpha \times \text{BM25}_{norm} + (1 - \alpha) \times \text{TF\text{-}IDF}_{norm}$$

- Default: **α = 0.6** (60% BM25, 40% TF-IDF)
- Adjustable via slider in the UI

## ⚙️ How the Engine Works

```
Query
  │
  ▼
Tokenize → lowercase, remove punctuation, stopwords, suffix stripping
  │
  ├──► TF-IDF Index → cosine similarity score
  │
  └──► BM25 Index  → probabilistic score
              │
              ▼
         (Hybrid: normalize + weighted sum)
              │
              ▼
         Ranked results sorted by final score
```

### Tokenizer Pipeline
1. Lowercase
2. Remove URLs and HTML tags
3. Remove punctuation
4. Remove stopwords (built-in list, no NLTK download needed)
5. Simple suffix stripping (`-ing`, `-ed`, `-tion`, `-er`, `-ly`, etc.)
